### （1）题目白话文解释

题目的核心诉求是：给定几种不同面值的硬币（且每种面值的硬币数量不受限制），以及一个目标总金额。要求计算出，**最少需要多少枚硬币**才能正好凑出这个总金额。如果利用给定的这些硬币怎么凑都凑不出这个精确的总金额，则要求返回 `-1`。

**生僻/关键概念解释：**
* **硬币数量无限**：意味着同一个面值的硬币可以被反复多次使用。例如，如果存在面值 `2` 的硬币，为了凑出金额 `4`，可以使用两枚面值 `2` 的硬币。
* **最少的硬币个数**：这是最终的优化目标。在所有能够刚好凑成目标金额的硬币组合中，只选取硬币总数量最少的那一种情况。

### （2）代码解题思路来源

这段代码采用的是**动态规划（Dynamic Programming）**算法，并且是经典的**完全背包问题**的一种变体形式。

解题思路从最基础的极端情况开始，逐步推导出复杂的目标情况。核心思想是建立一个一维数组 `f` 来保存中间状态，其中 `f[c]` 代表“凑出金额 `c` 所需的最少硬币个数”。

1.  **确定基础状态**：凑出金额 `0` 显然需要 `0` 枚硬币，这是一个确定的基准点。对于其他未知金额，初期统一假定为无穷大（`inf`），代表尚未找到凑齐的方案。
2.  **状态转移（递推逻辑）**：依次考察每一种面值的硬币 `x`。当手中拿到面值为 `x` 的硬币时，去尝试更新从金额 `x` 到目标金额 `amount` 之间的所有组合状态。
3.  **局部最优选择**：在考察目标金额 `c` 时，面临一种选择——是否使用当前这枚硬币 `x`。
    * 如果不使用，所需的硬币数就是之前已经计算出的 `f[c]`。
    * 如果决定使用一枚面值为 `x` 的硬币，那么剩下的金额缺口变成了 `c - x`。此时所需的硬币总数等于“凑出剩余金额 `c - x` 的最少硬币数（即 `f[c - x]`）”加上刚刚决定使用的这一枚硬币（即 `+ 1`）。
    * 将这两种选择的结果进行对比，只保留数值较小的那个，从而保证数组里存的始终是“最少个数”。
4.  通过多轮遍历更新，数组不断被刷新，最终数组末尾的元素 `f[amount]` 就会固化为目标金额的全局最优解。

### （3）带有注释的代码

```python
class Solution:
    def coinChange(self, coins: List[int], amount: int) -> int:
        # 创建并初始化动态规划数组，长度为 amount + 1，除 f[0] 外均设为无穷大
        # 对于其他未知金额，初期统一假定为无穷大inf，代表尚未找到凑齐的方案
        f = [0] + [inf] * amount
        # 外层循环：逐一遍历给定的每一种面值的硬币
        for x in coins:
            # 内层循环：遍历所有可能使用该硬币的金额区间（从硬币面额自身一直到总金额）
            for c in range(x, amount + 1):
                # 状态转移方程：在“原有最少硬币数”和“使用一枚当前硬币后的硬币数”中取最小值
                # f[c] 代表凑出金额 c 所需的最少硬币个数
                f[c] = min(f[c], f[c - x] + 1)
        # 提取数组最后一位，即为凑成目标金额的最少硬币数
        ans = f[amount]
        # 判断最终结果：如果小于无穷大，说明存在有效组合；否则返回 -1 表示无解
        return ans if ans < inf else -1
```

6.  `f[c] = min(f[c], f[c - x] + 1)`：这是最核心的状态转移执行步骤。计算出“使用一枚硬币 `x` 后的总数（即先查阅凑齐金额 `c-x` 需要的最少硬币数 `f[c-x]`，再加上刚用的这一枚 `+ 1`）”，将其与原本记录在案的 `f[c]` 进行对比，调用 `min()` 函数选取两者间的较小值，并重新写回 `f[c]` 中。
7.  `ans = f[amount]`：双层循环完全结束后，取出目标总金额 amount 对应的状态值 `f[amount]`，并赋值给独立变量 `ans`。
8.  `return ans if ans < inf else -1`：对结果进行最终判定与输出。如果 `ans` 的值小于正无穷，代表中途被成功更新过，找到了凑齐的方案，直接返回 `ans`；若仍等于正无穷，代表所有的硬币组合都无法填满该金额，按题目要求返回 `-1`。

```python
`ans = f[amount]`,  `return ans if ans < inf else -1`, 这个是如何保证，ans里面对应的所有[inf] * amount都是可以更新的？
代码其实**根本没有保证** `f` 数组里的所有 `inf` 都会被更新。
恰恰相反，如果某个金额无论如何都凑不出来，那么它在 `f` 数组中对应的值就会**从头到尾一直保持为 `inf`**。
为什么有些 `inf` 永远不会被更新？
状态转移方程是 `f[c] = min(f[c], f[c - x] + 1)`。
假设当前想凑金额 `c`，打算使用一枚面值为 `x` 的硬币，那么就需要先去查“凑出剩余金额 `c - x` 需要几枚硬币”，也就是去查 `f[c - x]` 的值。
* **如果剩余金额 `c - x` 本身就凑不出来**：那么 `f[c - x]` 记录的值就是 `inf`。
* **数学运算规则**：在 Python（以及大多数数学体系）中，无穷大加一依然是无穷大，即 `inf + 1 = inf`。
* **比较结果**：`min(inf, inf + 1)` 的结果依然是 `inf`。
这意味着，如果一个状态所依赖的前置状态是inf，那么当前状态也会继续保持为inf。

反例演示（凑不出的情况）
假设：
* 给定硬币：`coins = [2]` （手里只有面额为 2 的硬币）
* 目标金额：`amount = 3`
```

| **初始状态** | 无 | 无 | 无 | `f = [0, inf, inf, inf]` |
| **处理面额2**| `x = 2` | `c = 2` | `f[2] = min(inf, f[0]+1)` <br> `= min(inf, 0+1) = 1` | `f = [0, inf, 1, inf]` |
| **处理面额2**| `x = 2` | `c = 3` | `f[3] = min(inf, f[1]+1)` <br> `= min(inf, inf+1) = inf` | `f = [0, inf, 1, inf]` |
| **循环结束** | 无 | 无 | 取出 `ans = f[3]` | `ans = inf` |

1.  金额 1 无法用 2 元硬币凑出，所以 `f[1]` 永远是 `inf`。
2.  当尝试凑金额 3 时，需要依赖 `f[3-2]` 也就是 `f[1]`。因为 `f[1]` 是 `inf`，导致 `f[3]` 也无法得到有效的数字，最终保持为 `inf`。

1.  判断 `ans < inf` 是否成立？
2.  代入发现 `inf < inf` 显然**不成立**（假）。
3.  于是执行 `else` 后面的逻辑，返回 `-1`。
